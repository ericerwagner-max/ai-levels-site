Concept 1

An LLM is a very good autocomplete

It got great at one trick: given some text, predict what comes next — one piece at a time. It doesn't look anything up. Press play and watch it rank the next word.

Concept 2

A chatbot talks. An agent does.

Hand the same model a goal, a set of tools, and permission to loop — and it stops describing the work and starts doing it. Watch the loop run.

Where AI already works in shoulder & elbow

This isn't hypothetical. Real, published work in our field — today.

Imaging & diagnosis
96–97%

Deep-learning detection of proximal humerus fractures on radiographs — matching shoulder specialists and beating generalists on 3- & 4-part fractures.

Outcome prediction
~85–94%

ML predicts pain, function & ROM after anatomic and reverse TSA from 5,700+ cases — with a feature set simple enough to use live in clinic.

NLP for registries
Free-text → data

Natural-language processing extracts outcomes from op notes and clinic letters, populating registries from charts you already have.

Literature & writing
Draft, not decide

LLMs screen citations and draft methods/limitations — but exam accuracy still trails residents, so every claim needs verification.

Sources: Chung et al., Acta Orthop 2018 & external-validation cohorts 2024–25; Kumar et al. minimal-feature TSA model; systematic reviews of ML & LLMs in orthopaedic surgery, 2024–25.

Mostly free · start tonight

Three moves to get started

You don't need a terminal or any code. Each move adds one capability — copy a starter prompt and try it.

1 · Seed it with you

Start a Project. Add your CV, 2–3 papers, and your goals. Now it answers as if it knows your work.

2 · Give it hands

Add connectors — PubMed, Drive, calendar. Answers come back with real, clickable citations.

3 · Encode a skill

Write your best workflow once. The agent runs it the same high-quality way every time.

A
Seed your Project — paste this into a fresh Project's custom instructions, then upload your CV and a few PDFs.
You are my research co-pilot in shoulder & elbow surgery. I've uploaded my CV and several of my papers — read them to learn my methods, my cohorts, my outcome measures, and my writing voice. Ground every factual claim in an uploaded file or a real citation. If you're unsure or can't verify something, say so — never invent a reference.
B
Run a real workflow — drop a folder of papers and ask for a verifiable summary + first draft.
Here's a folder of papers on reverse TSA for cuff-tear arthropathy. For each: pull the design, n, follow-up, key outcomes, and complications into a table. Verify every PMID. Then draft a 150-word intro paragraph in my voice, citing only papers from this folder.
C
Turn it into a skill — once a workflow works, ask it to save the steps so it runs identically next time.
That worked well. Write these exact steps down as a reusable "paper-ingest" skill: read a PDF → extract design/n/outcomes → verify references → output a one-page knowledge node. Use it whenever I drop a new paper.

The personal-OS core (a Project + memory) is free. The agentic workspace that reads whole folders and creates files is a paid feature.

The one rule that never moves: no patient information. Ever.

A personal account is not HIPAA-covered (no BAA). Use only your own scholarly materials — never patient data or institutional systems. Clinical or identifiable work belongs in an institution-approved, BAA-backed tool. And never paste passwords or keys.
